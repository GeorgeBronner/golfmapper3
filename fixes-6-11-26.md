# Fixes — June 11, 2026

Full frontend + backend review findings, tracked as they get fixed on the `june26-fixes` branch.

Status legend: ✅ done · 🚧 in progress · ⬜ not started · ⏸ deferred (with reason)

## P0 — Requested feature

- ✅ **One course per user, one year.** A user can only add a course once (with a single year).
  Backend: `add_course` rejects a second add of the same course (any year) with 409, and validates
  the course ID exists (404 if not — previously orphan rows were silently created on SQLite).
  Model: unique constraint tightened from `(user_id, course_id, year)` to `(user_id, course_id)`
  (applies to fresh databases; existing DBs are covered by the application-level check).
  Frontend: `CourseForm` shows "You have already added this course." on 409 and a clear message on 404.

## P1 — Bugs

- ✅ **`readall_ids_w_year` mutates shared ORM objects** (`user_courses.py`). SQLAlchemy's identity
  map returns the same `Courses` instance for repeated rows, so duplicate plays collapsed to the
  last year and shared a `user_course_id` (duplicate React keys, wrong-row year edits). Fix: build
  a response model per row instead of mutating ORM objects.
- ✅ **Rate limiting keyed on proxy IP** (`limiter.py`). Behind nginx/docker every request shares the
  proxy address, so the 10/min login limit was global across all users. Fix: key on the first
  `X-Forwarded-For` hop when present.
- ✅ **Unknown API paths return HTTP 200** (`main.py` SPA catch-all). `/api/v1/<anything>` unmatched
  GETs returned `{"detail": "Not Found"}` with status 200. Fix: real 404 response.
- ⬜ **Frontend test suite 100% failing** (`localStorage` undefined). Duplicated/conflicting test
  config: `vite.config.js` `test` block (happy-dom) vs `vitest.config.js` (jsdom). Fix: single
  config, suite green.
- ⬜ **Sentry misconfig** (`index.js`): `tracePropagationTargets` contained the DSN instead of the
  API origin. Plus `vite.config.js` ships the Sentry build plugin with placeholder `org: "your-org"`
  — removed.
- ✅ *(resolved by P0 design)* **Delete-by-course_id removed an arbitrary year row** when a course
  had multiple year entries. With one entry per course, delete by `course_id` is unambiguous.

## P2 — Backend quality

- ✅ **Admin auth copy-pasted 11×** (`admin.py`, `course_requests.py`). Replace with a shared
  `require_admin` dependency; wrong role now returns **403** (was 401); drop the dead
  `if user is None` checks (`get_current_user` raises before ever returning None).
- ✅ **`get_db` defined three times** (`dependencies.py`, `auth.py`, `password_reset.py`). Single
  definition in `database.py`, re-exported where tests import it.
- ✅ **Tests require `SECRET_KEY_AUTH` exported manually.** Set a default in the test package so
  `pytest` works out of the box.
- ✅ **`pyproject.toml` hygiene**: `pytest`/`pytest-asyncio` were production dependencies; `ruff`
  duplicated in `[project.optional-dependencies]` and `[dependency-groups]`; ruff
  `target-version = "py313"` contradicted `requires-python = ">=3.14"`.
- ✅ **Password rule inconsistency**: min 6 chars in `users.py` vs 8 everywhere else. Unified to 8.
- ✅ **`generate_all_users_map` N+1 query** (`map.py`): one query per active user. Fix: single join,
  grouped in Python.
- ✅ **Dev scripts inside the app package**: `app/manual_GPS_edit.py` (runs `input()` at module
  level — importing it hangs the process) and `app/utils/map_course_entry.py` (standalone FastAPI
  tool). Moved to `backend/scripts/`.

## P2 — Frontend quality

- ⬜ **Dead components**: `GarminCourses.jsx`, `GarminCourseRow.jsx`, `LoginHeader.jsx`,
  `Footer.jsx` — referenced by nothing. Removed.
- ⬜ **Unused dependencies**: `@faker-js/faker` (was a *production* dep), `web-vitals`, `js-cookie`.
  Removed.
- ⬜ **`config.js` no fallback** — unset `VITE_BACKEND_SERVER_IP` produced `http://undefined:80`.
  Fix: default to `window.location.origin` (the backend serves the frontend in prod).
- ⬜ **`CourseList` fetch has no error handling** — failures showed "0 courses tracked" silently.
  Added an error state.
- ⬜ **`AuthProvider` sets global axios defaults nothing uses** — the app talks through the `api`
  instance which reads localStorage itself. Removed the dead global-axios coupling.
- ⬜ **`vite.config.js` cruft**: leftover CRA-era `define` of `process.env.REACT_APP_*`, redundant
  babel JSX plugin, placeholder Sentry plugin, plus redundant `.babelrc`. Removed; build verified.

## Deferred (not in this branch)

- ⏸ **Alembic migrations** — worth doing before the next schema change, but a project-level
  decision; `create_all` can't alter existing tables (note: the new unique constraint in `models.py`
  only applies to fresh DBs for this reason).
- ⏸ **Swap `python-jose` → `PyJWT`** — jose is unmaintained (2024 CVEs were patched, but PyJWT is
  the maintained choice). Mechanical change; kept out to limit this branch's blast radius.
- ⏸ **CORS origins → env-driven** — deployment-specific; the hardcoded list includes `http://`
  production variants and stale IPs that should be reviewed.
- ⏸ **`CourseSearch` loads all courses client-side** — fine at current scale; switch to the
  existing paginated endpoint if the table grows.
- ⏸ **Map components use `doc.write()`** — `srcDoc` is the cleaner approach; cosmetic.
- ⏸ **`ProtectedRoute`/`AdminRoute` read localStorage directly** instead of `useAuth` — works,
  consolidation is a nice-to-have.

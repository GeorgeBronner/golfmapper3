## P0 ‚Äî Requested feature

- ‚úÖ **One course per user, one year.** A user can only add a course once (with a single year).
  Backend: `add_course` rejects a second add of the same course (any year) with 409, and validates
  the course ID exists (404 if not ‚Äî previously orphan rows were silently created on SQLite).
  Model: unique constraint tightened from `(user_id, course_id, year)` to `(user_id, course_id)`
  (applies to fresh databases; existing DBs are covered by the application-level check).
  Frontend: `CourseForm` shows "You have already added this course." on 409 and a clear message on 404.

## P1 ‚Äî Bugs

- ‚úÖ **`readall_ids_w_year` mutates shared ORM objects** (`user_courses.py`). SQLAlchemy's identity
  map returns the same `Courses` instance for repeated rows, so duplicate plays collapsed to the
  last year and shared a `user_course_id` (duplicate React keys, wrong-row year edits). Fix: build
  a response model per row instead of mutating ORM objects.
- ‚úÖ **Rate limiting keyed on proxy IP** (`limiter.py`). Behind nginx/docker every request shares the
  proxy address, so the 10/min login limit was global across all users. Fix: key on the first
  `X-Forwarded-For` hop when present.
- ‚úÖ **Unknown API paths return HTTP 200** (`main.py` SPA catch-all). `/api/v1/<anything>` unmatched
  GETs returned `{"detail": "Not Found"}` with status 200. Fix: real 404 response.
- ‚úÖ **Frontend test suite 100% failing** (`localStorage` undefined). Three stacked causes:
  (1) conflicting test config ‚Äî `vite.config.js` `test` block (happy-dom) vs `vitest.config.js`
  (jsdom): removed `vitest.config.js`; (2) Node 22+'s experimental `localStorage` global (undefined
  without `--localstorage-file`) shadows the DOM environment's implementation: test setup now
  installs an in-memory Storage; (3) a stale assertion matched the newer "Forgot password?" link.
  Also renamed JSX-containing `.js` files (`index`, `router`, `App.test`) to `.jsx`, which let the
  esbuild loader hacks in `vite.config.js` go away. Suite green (4/4).
- ‚úÖ **Sentry misconfig** (`index.js`): `tracePropagationTargets` contained the DSN instead of the
  API origin. Plus `vite.config.js` ships the Sentry build plugin with placeholder `org: "your-org"`
  ‚Äî removed.
- ‚úÖ *(resolved by P0 design)* **Delete-by-course_id removed an arbitrary year row** when a course
  had multiple year entries. With one entry per course, delete by `course_id` is unambiguous.

## P2 ‚Äî Backend quality

- ‚úÖ **`get_db` defined three times** (`dependencies.py`, `auth.py`, `password_reset.py`). Single
  definition in `database.py`, re-exported where tests import it.
- ‚úÖ **Tests require `SECRET_KEY_AUTH` exported manually.** Set a default in the test package so

- ‚úÖ **Password rule inconsistency**: min 6 chars in `users.py` vs 8 everywhere else. Unified to 8.
- ‚úÖ **`generate_all_users_map` N+1 query** (`map.py`): one query per active user. Fix: single join,
  grouped in Python.
- ‚úÖ **Dev scripts inside the app package**: `app/manual_GPS_edit.py` (runs `input()` at module
  level ‚Äî importing it hangs the process) and `app/utils/map_course_entry.py` (standalone FastAPI
  tool). Moved to `backend/scripts/`.

## Round 2 (second pass on the same branch)

- ‚úÖ **Swap `python-jose` ‚Üí `PyJWT`** ‚Äî jose is unmaintained (2024 CVEs were patched, but PyJWT is
  the maintained choice). `auth.py` and `test_auth.py` now use PyJWT; the test secret was lengthened
  to 32+ bytes (PyJWT warns on shorter HS256 keys ‚Äî check the production `SECRET_KEY_AUTH` length).
- ‚úÖ **CORS origins documented and env-overridable** ‚Äî the list in `config.py` is grouped and
  commented, and can be overridden per deployment via the `CORS_ORIGINS` env var (JSON list).
  Removed only the two `golf.bronnerapp.com:80` entries ‚Äî a browser Origin header never carries a
  default port, so they could never match. Verified against `homelab/oracle/apps.md`: the prod
  origin (`https://golf.bronnerapp.com`), stage, and the Oracle box IP entries are all retained.
- ‚úÖ **Bundle code-splitting** ‚Äî admin pages and the leaflet/bootstrap-heavy course-edit pages are
  now `React.lazy` routes. Main chunk: 920 kB ‚Üí 689 kB (gzip 291 kB ‚Üí 220 kB).
- ‚úÖ **Map components use `doc.write()`** ‚Äî both map iframes now use `srcDoc`.
- ‚úÖ **`ProtectedRoute`/`AdminRoute` read localStorage directly** ‚Äî both now read the token via
  `useAuth`.
